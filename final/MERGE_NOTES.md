# MERGE_NOTES — `final/` 통합 근거

PM(병합 담당)이 `강주현/`, `김현수/`, `박이완/`, `이화목/`, `임준혁/`, `한태우/` 6명의 독립 구현을
spec.md 6번 병합 기준(5개 항목)으로 비교한 뒤, 항목별로 가장 나은 구현을 조합한 결과입니다.
전체 감사는 각 폴더의 `main.py`/`report.md`를 직접 읽고, 실제 이미지·HTML을 열어 라벨을
확인하고, `na_values`/`stratify`/`equal_var` 등 정확한 코드 값을 인용해 검증했습니다.

## 항목별 채택 근거 (spec.md 6번 기준)

1. **결측치·중복 처리 → 강주현 기반(+임준혁 아이디어 가미)**: `na_values="?"`(공백 없음)를
   `skipinitialspace=True`와 함께 정확히 사용했고, 대체 전/후 결측치 수를 실제로 출력해 최빈값
   대체가 진짜 일어났음을 증명했습니다. `final/main.py`에서는 여기에 임준혁의 "Pandas·Polars
   교차검증 assert" 아이디어를 추가해, 두 라이브러리의 shape·결측치·중복행 수가 완전히
   일치하는지 코드 레벨에서 강제 검증하도록 강화했습니다(콤마+공백으로 인한 Polars 숫자
   컬럼 String 오분류, trailing blank line으로 인한 행 수 불일치를 모두 예방).
2. **시각화 → 임준혁 기반**: Seaborn 1장에 income별 근무시간 박스플롯(t-test 대상 변수의
   시각적 근거)과 상관계수 히트맵(spec 상관표의 시각화)을 2-패널로 결합해 정보 밀도가 가장
   높았고, Plotly는 학력×income 그룹 막대 차트로 두 범주형·수치형 변수를 동시에 보여줘
   인사이트가 가장 풍부했습니다. 실제 PNG·HTML을 열어 제목·축 라벨이 모두 존재함을 확인했습니다.
3. **통계 검정 → 강주현 기반**: 계산 로직(`ttest_ind(..., equal_var=False)`, 그룹 순서)은
   전원 동일하게 정확했지만, report.md의 "핵심 인사이트" 문장에서 어느 그룹이 더 긴 근무시간을
   갖는지를 `higher_group = ">50K" if over_mean > under_mean else "<=50K"`처럼 **실제 평균을
   비교해 동적으로 결정**하는 방식을 채택했습니다. 데이터가 바뀌어도 문장이 저절로 맞게
   갱신되어, 하드코딩으로 인한 불일치 위험(한태우 사례 참고)을 원천 차단합니다.
4. **ML Pipeline → 강주현 기반 골격 + 임준혁 방어 로직**: `RandomForestClassifier` +
   `ColumnTransformer`/`Pipeline`로 6명 중 최고 accuracy(0.8657)를 냈고, `stratify=y`·
   `test_size=0.2`·`random_state=42`를 모두 준수하며 `fnlwgt`/`income` 누수가 없었습니다.
   여기에 임준혁의 `SimpleImputer`(수치형 median, 범주형 most_frequent)를 파이프라인 내부에
   방어적으로 추가해, feature 서브셋을 다른 맥락에서 재사용해도 안전하도록 견고성을
   높였습니다. 6명 전원이 누락했던 "joblib 재로딩 검증"도 `np.allclose`로 새로 추가했습니다
   (n_jobs=-1 병렬 학습은 재로딩 후 부동소수점 미세 오차가 날 수 있어 `==` 대신 사용).
5. **report.md 자동화 → 강주현 기반**: f-string 기반으로 완전히 스크립트에서 생성되고,
   "핵심 인사이트" 섹션의 모든 문장이 실제 계산값에서 동적으로 도출됩니다(어느 그룹이
   더 긴지, 어느 변수쌍의 상관이 가장 큰지 모두 `idxmax()`/조건문으로 계산). `tabulate`
   패키지 의존성 없이 직접 마크다운 표를 생성하는 `_df_to_markdown()`도 그대로 채택해,
   한태우 사례처럼 하드코딩된 수치가 실제 값과 어긋나는 문제를 원천 차단했습니다.

## `final/` 구조에서 달라진 점

- 팀원 폴더는 각자 `output/` 하위(박이완)거나 루트 직속(나머지)이라 산출물 위치가
  제각각이었지만, `final/`은 spec.md 5번 규칙대로 5개 산출물(`eda_chart_seaborn.png`,
  `eda_chart_plotly.html`, `model.joblib`, `report.md`, `main.py`)을 모두 `final/` 루트에 둡니다.
- 데이터 소스는 네트워크 의존성을 없애기 위해 박이완이 커밋해둔 `data/adult.data`를
  `final/data/adult.data`로 복사해 로컬 파일에서 읽도록 통일했습니다(원본 URL과 동일한
  UCI Adult 데이터셋).
- 강주현의 "추가(선택) 시각화" 2종(그룹비교 박스플롯, 분포 히스토그램)은 임준혁의 2-패널
  Seaborn 차트에 이미 박스플롯이 포함되어 중복이므로 `final/`에서는 필수 2종(Seaborn 1 +
  Plotly 1)만 남겨 산출물을 간결하게 유지했습니다.
